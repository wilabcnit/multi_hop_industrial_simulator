---

## Reporting Bugs
- Open a [GitHub Issue](../../issues) and clearly describe:
  - Steps to reproduce the bug  
  - Expected behavior and observed behavior  
  - Environment (OS, Python version, compiler version, etc.)

---

## New Features
- Before implementing a new feature, please open an Issue to discuss it.  
- Pull Requests (PRs) introducing major changes without prior discussion may not be accepted.

---

## Pull Requests
1. Fork this repository.  
2. Create a branch with a descriptive name (e.g., `feature/new-scheduler`).  
3. Submit your PR to the `main` branch, clearly explaining the changes made.

---

## Code Style
- Follow [PEP 8](https://peps.python.org/pep-0008/) guidelines.  
- Use docstrings for functions and classes.  
- Add test files whenever possible.
